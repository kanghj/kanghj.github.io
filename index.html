Software (including AI) is eating the world but there is an incoming software apocalypse. We desperately need better ways of understanding software systems. My research aims to develop approaches to mine abstractions to address important Software Engineering problems. In my PhD, my research focused on bugs and vulnerabilities related to APIs and third-party components, e.g. libraries.

[Research Statement][Teaching Statement]

Mining abstractions. I am interested in mining task-specific abstractions (e.g. specifications, rules, features) of programs.

Related to the abstractions of programs used in Machine Learning for Software Engineering, we analyzed the generalizability of a representative token-based embeddings [ASE 2019]. For filtering false positives from static analysis, we addressed methodological issues [ICSE 2022 (Poster)], and used language models for refining call graphs [FSE 2022].

We mined behavioral models from traces generated using search-based test generation [TOSEM 2020] and inferred sandbox rules (specified as Alloy models) for an IoT system [ICST 2021].

Library and API usage. Modern software development involves the heavy use of third-party systems (libraries and their APIs). For library vulnerabilities, we have:

  1. employed a transformer model and a zero-shot approach to automatically identify the libraries described in NVD entries (have been deployed by our industrial partners) [ICPC 2022, ICSE 2023],
  2. identified vulnerabilities by analyzing the commit history of open source projects [SANER 2022, ICSE 2022 Demo, TSE], and
  3. generated test cases for programs for assessing the exploitability of vulnerabilities in libraries they imported[ISSTA 2022].
For APIs, we developed an API misuse detector that uses active learning for learning subgraph patterns from programs across GitHub [TSE 2022].

Program transformation. We developed Coccinelle4J [ECOOP 2019] for program transformation of Java programs, extending the Coccinelle program matching and transformation tool. Coccinelle is widely adopted in the Linux kernel and C systems software. To build better tools, there are many lessons we can learn from Coccinelle.

Building on Coccinelle4J, we have worked on the migration of deprecated Android API methods [ICSME 2019, ICPC ERA, EMSE 2022]. Other work include machine learning on code changes/commits [ICSE 2020, SANER 2022]

Better research on program transformation will help automate more programming.
