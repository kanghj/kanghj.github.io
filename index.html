Hong Jin Kang

PhD Student at Singapore Management University

Many aspects of Software Engineering interests me, but my primary interest is automated software engineering. Software is eating the world, and there is a coming software apocalypse that we should try to prevent. If we have better software, the world will be a safer place. Better software can be built if there were better developer tools. My goal is to help, improving developer productivity by taking best ideas from machine learning, data mining, and metaheuristic search techniques.

Mining specifications. My work so far has focused on mining rules and specifications about software. We have looked into better ways to mine automaton models of source code [TOSEM 2020]. These models represent sequential constraints/protocols of usage. At the moment, I am also studying other representations of source code, as well as rules in less traditional software systems.

Program transformation. I am also very interested in program transformation. We developed Coccinelle4J [ECOOP 2019], building on top of the Coccinelle program matching and transformation tool. Coccinelle is unique in many ways and it is widely adopted in the Linux kernel and C systems software community. Key to Coccinelle's success is how it allows for the expression of constraints over control-flow. I think there are many lessons and insights we can take from Coccinelle to build better dev tools. A lot of our work have been on the Android ecosystem, including studies on how to infer rules to migrate usages of deprecated API methods with their replacement methods [ICSME 2019, ICPC 2020 ERA].

Representation of software. To better understand software, I am also interested in how best to represent a piece of program. For instance, in recent years, there has been growing interest in using techniques from machine learning, such as embedding entities into a low dimensional vector space. In this area, we investigated the generalizability of a representative token-based embeddings [ASE 2019], and proposed embeddings for representing code changes [ICSE 2020].
