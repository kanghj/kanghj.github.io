Hong Jin Kang

PhD Candidate at Singapore Management University

I am a PhD candidate at Singapore Management University under the SOAR group. I am very fortunate to be advised by Prof. David Lo and to collaborate with many brilliant and talented researchers.

Software is eating the world but there is an incoming software apocalypse. We desperately need better ways of understanding and engineering software systems. My research goal is to develop approaches to design and mine abstractions to address important Software Engineering problems.

[Resume] [Google Scholar] [DBLP]

Mining abstractions. My primary interest is on mining task-specific abstractions (e.g. specifications) of programs.

We have mined behavioral models using search-based test generation [TOSEM 2020] and inferring sandbox rules specified as Alloy models for an IoT software system [ICST 2021].

We have analyzed the features used in some Machine Learning for Software Engineering approaches, including the generalizability of a representative token-based embeddings [ASE 2019], and addressed methodological issues (in the computation of some features) for filtering warnings from static analyzers ([ICSE 2022]).

We are interested in helping developers use third-party systems (libraries and APIs). For managing vulnerabilities, we have:

  1. extracted features using pre-trained transformer models to automatically identify affected libraries from NVD entries (to be deployed by our industrial partner, Veracode, [ICPC 2022]),
  2. combined multiple information sources in identifying commits that fix vulnerabilities ([SANER 2022]), and
  3. developed abstractions of vulnerable behavior for generating test cases that reveal usages of vulnerable library functionabilities ([ISSTA 2022]).
For managing usage of APIs, we developed an API misuse detector that uses active learning for learning subgraph patterns from programs across GitHub ([TSE 2022]).

Better abstractions will enable better developer tools.

Program transformation/code commits. We developed Coccinelle4J [ECOOP 2019] for program transformation of Java programs, extending the Coccinelle program matching and transformation tool. Coccinelle is widely adopted in the Linux kernel and C systems software. To build better tools, there are many lessons we can learn from Coccinelle.

Building on Coccinelle4J, we have worked on the migration of deprecated API methods in Android ([ICSME 2019, ICPC 2020 ERA],EMSE 2022). Other work include machine learning for code changes/commits ([ICSE 2020], [SANER 2022] ) Better research on program transformation will help automate more programming.
