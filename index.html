Software (including AI) is eating the world but there is an incoming software apocalypse. We desperately need better ways of understanding software systems. My research aims to develop approaches that empowers human users to address important Software Engineering problems. To do so, automated approaches have to use approriate task-relevant abstractions. In my PhD, my research focused on bugs and vulnerabilities related to APIs and third-party components, e.g. libraries.

Mining abstractions.

For AI4SE, we analyzed the generalizability of a representative embeddings model [ASE 2019]. For filtering false alarms from static analyzers, we addressed methodological issues [ICSE 2022 (Poster)], and subsequently reestablished state-of-the-art results [TSE 2023]. We also used large language models for pruning false positives from call graphs [FSE 2022].

We mined models by exploring program behaviors using evolutionary test generation [TOSEM 2020], and inferred sandbox rules for an IoT system [ICST 2021].

Library and API usage. Modern software development heavily uses third-party systems (e.g., libraries). For library vulnerabilities, we have built systems that have been deployed by our industrial partners. We have :

  1. employed large language models and a zero-shot approach to automatically identify the libraries described in NVD entries [ICPC 2022, ICSE 2023],
  2. helped in monitoring open source repositories to detect unreported vulnerabilities [SANER 2022, ICSE 2022, TSE 2023], and
  3. used test generation for assessing the possible impact and reachability of library vulnerabilities[ISSTA 2022].
For API usage, we developed an API misuse detector using active learning to learn from programs on GitHub [TSE 2022]. We fuzzed the APIs of TensorFlow and PyTorch (resulting in over 20 CVE IDs) [under submission to TSE].

Program transformation. We developed Coccinelle4J [ECOOP 2019] for program transformation of Java programs, extending the Coccinelle program matching and transformation tool. Coccinelle is widely adopted in the Linux kernel and C systems software. To build better tools, there are many lessons we can learn from Coccinelle.

Using Coccinelle4J, we have worked on the automated migration of deprecated Android API methods [ICSME 2019, ICPC ERA, EMSE 2022]. Other work include machine learning on code changes/commits [ICSE 2020, SANER 2022]
